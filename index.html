Journey

My developer journey began with the general exploration of programming languages in school/college such as Python, C#, HTML/CSS/JavaScript and SQL
Curiosity led me to learning C++ and more advanced Python myself at home.



Project 1 - Godot Game

I then explored game development using the Godot game engine, and using a scripting language similar to Python called GDScript.
After spending months learning the engine and building test games, I developed my own game over the course of 3 months and released it to itch.io.

This was my first developer experience. It's where I first found the enjoyment in the process of learning new concepts, implementing them and testing.
More importantly, I learnt the importance of file organisation after my project grew to contain multiple different scripts and scenes




Project 2 - Blender 3D Model

Following this, I learnt 3D Modelling using Blender, and made my own version of a popular Blender project: The Blender Donut.


This project gave me the opportunity to test my creativity. Learning a whole new skill and software like this expanded my problem solving ability and demonstrated my proficiency in different software and digital skills.





Project 3 - Unity Game

During our A Level Computer Science course, we developed a game as part of our NEA coursework. Across 4 months I developed and documented the development of another 2d top-down tank shooter game, this time using the Unity game engine and the scripting language C#.
We also learnt the basics of HTML, CSS and JavaScript.
This sparked the passion in me once again, and as of recently I have further developed my skills in these languages. (All the project files from below are accessible from my Github)



Project 4 - Rock Paper Scissors Game (HTML/CSS/JavaScript)



I began by following a tutorial for this game before adding basic customisation.
Although the code used was fairly simple and similar to that of the tutorial, this project was crucial in giving me the opportunity to experiment with these programming languages and making me comfortable with HTML, CSS and JavaScript.




Project 5 - Matching Pairs Game (HTML/CSS/JavaScript)



This project was slightly more difficult and I began working on this project after finding a tutorial for it online.
Not long after starting however, my algorithmic skills from previous projects kicked in and I found I could complete the rest of the project without using the tutorial.
While my approach may not have been perfect or the most efficient, it was my first experience of writing a significant amount of code myself in these programming languages. Here, I began understanding the debugging process when logical or syntax errors inevitably arise.




Project 6 - Calculator (HTML/CSS/JavaScript)



This project was created with extensive use of all 3 languages, and was created without the use of any tutorial.
The JavaScript aspect demanded a great amount of logical thinking and validation checks/testing before I was satisfied.
Styling this page gave me the oppurtunity to learn more about CSS and how it can be used to make a website appealing. I was able to use my creativity to experiment with making the site stand out and look good.




Project 7 - Date Picker (HTML/CSS/JavaScript)



This project was also created without the use of any tutorial.
After seeing this idea on Developer Roadmaps, I saw it as an opportunity to further develop my skills in HTML, CSS and JavaScript - specifically practicing how to style grids in CSS and how to use the "Date()" function appropriately in JavaScript.




Project 8 - Quiz website (full-stack - HTML/CSS/JavaScript, PHP, MySQL

I am currently working on a full-stack project to test my ability.
it will include multiple elements: an interface for my quiz game, a good, consistent styling for my website and a relational database with tables for questions, accounts and quiz history.
For this, I am using HTML, CSS and JavaScript for the front-end, PHP for the back-end and MySQL for the database.
